# 贡献指南

感谢您对选书神器项目的兴趣！我们欢迎各种形式的贡献，无论是代码、文档、设计还是想法。

## 🚀 如何开始

### 1. 了解项目

在开始贡献之前，请仔细阅读以下文档：
- [README.md](README.md) - 项目概述和技术架构
- [项目介绍.md](项目介绍.md) - 详细的项目介绍
- [MVP设计方案.md](MVP设计方案.md) - 当前开发计划

### 2. 环境准备

```bash
# 克隆项目
git clone https://github.com/MRYGP/shuanshushengqi.git
cd shuanshushengqi

# 安装依赖（如果有）
pip install -r requirements.txt  # 后端
npm install                    # 前端
```

### 3. 开发流程

1. **创建功能分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **提交代码**
   ```bash
   git add .
   git commit -m "feat: 你的功能描述"
   ```

3. **推送分支**
   ```bash
   git push origin feature/your-feature-name
   ```

4. **创建 Pull Request**
   - 在 GitHub 上创建 PR
   - 详细描述你的更改
   - 等待代码审查

## 📝 提交信息规范

我们使用约定式提交格式：

```
<type>(<scope>): <subject>

[body]

[footer]
```

### 类型 (Type)
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或工具配置

### 示例
```
feat: 添加用户画像收集功能
fix: 修复推荐算法的冷启动问题
docs: 更新API文档
```

## 三、博主风格参数（第二层 - 核心差异化）

### 3.1 什么是"镜像博主"？

**核心思想**： 把博主的内容风格参数化，让推荐带有"博主的味道"

**类比**：

- Spotify：根据你听过的歌，推荐相似风格的新歌
- 我们：根据博主的内容风格，推荐匹配的书籍

### 3.2 风格参数设计（五个维度）

```json
{
  "creator_id": "mirror_yangtianzhen",
  "display_name": "杨天真风格",
  "style_weights": {
    "difficulty": {
      "入门": 0.40,
      "进阶": 0.40,
      "专业": 0.20
    },
    "tools": 0.50,        // 工具密度 (0-1，工具vs思维)
    "story": 0.50,        // 故事性 (0-1，案例vs理论)
    "minutes": [15, 35],  // 时长偏好 (分钟范围)
    "explore": 0.20       // 探索比例 (0-1，保守vs冒险)
  },
  "scenarios": [          // 优先场景
```

### 3.3 参数如何影响推荐？

| 维度     | 杨天真          | Olga姐姐        | 推荐差异         |

| 难度分布 | 入门40%/进阶40% | 入门50%/进阶35% | 杨推更进阶的书   |
| 工具密度 | 0.50（平衡）    | 0.60（偏工具）  | Olga推更多方法论 |
| 故事性   | 0.50（很强）    | 0.40（适度）    | 杨推更多案例书   |
| 时长偏好 | 15-35分钟       | 15-40分钟       | 相近             |

**实际效果**：

- 同样的用户画像

```python
def get_recommendations(user_profile, user_id=None, creator_style=None):
    """
    混合推荐系统
    
    Args:
        user_profile: 用户画像
        user_id: 用户ID（如果有历史）
        creator_style: 博主风格参数（可选）
    
    Returns:
        推荐书籍列表 + 推荐理由
    """
    
    # Step 1: 规则打分（60%权重）
    rule_scores = rule_based_scoring(user_profile, creator_style)
    
    # Step 2: 协同过滤（40%权重）
    if user_id and has_rating_history(user_id):
        cf_scores = collaborative_filtering(user_id)
    else:
        # 冷启动兜底
        cf_scores = cold_start_recommendations(
            popularity_weight=0.7,
            explore_weight=0.3
        )
    
    # Step 3: 混合得分
    final_scores = 0.6 * rule_scores + 0.4 * cf_scores
    
    # Step 4: MMR多样性重排
    diverse_recs = mmr_rerank(
        candidates=final_scores,
        lambda_param=0.7,
        top_k=5
    )
    
    # Step 5: 生成推荐理由
    recommendations = []
    for book in diverse_recs:
        reason = generate_explanation(
            book=book,
            user_profile=user_profile,
            creator_style=creator_style
        )
        recommendations.append({
            "book": book,
            "reason": reason,
            "confidence": book["score"]
        })
    
    return recommendations[:3]  # 返回Top3
```

### 3.2 规则打分算法

```python
def rule_based_scoring(user_profile, creator_style=None):
    """
    基于规则的打分
    
    维度：
    1. 主题匹配（0.5权重）
    2. 难度匹配（0.3权重）
    3. 风格匹配（0.2权重）
    4. 场景匹配（bonus）
    """
    
    scores = {}
    
    for book in book_library:
        score = 0.0
        
        # 主题匹配（0.5）
        if book["topic"] in user_profile["topics"]:
            score += 0.5
        
        # 难度匹配（0.3）
        if book["difficulty"] == user_profile["difficulty"]:
            score += 0.3
        elif is_adjacent_difficulty(book, user_profile):
            score += 0.15  # 相邻难度给一半分
        
        # 风格匹配（0.2）
        if book["style"] in user_profile["styles"]:
            score += 0.2
        
        # 场景匹配（bonus，最多+0.1）
        scenario_match = count_scenario_overlap(book, user_profile)
        score += min(scenario_match * 0.05, 0.1)
        
        # 博主风格加权（如果提供）
        if creator_style:
            score *= get_creator_weight(book, creator_style)
        
        scores[book["book_id"]] = score
    
    return scores
```

### 3.3 MMR多样性算法

```python
def mmr_rerank(candidates, lambda_param=0.7, top_k=5):
    """
    Maximal Marginal Relevance (MMR)
    
    核心思想：
    - 平衡相关性（relevance）和多样性（diversity）
    - λ=1.0：完全看相关性（可能推荐3本同类书）
    - λ=0.0：完全看多样性（可能推荐不相关的书）
    - λ=0.7：平衡点（推荐准确且有多样性）
    
    公式：
    MMR = λ * Relevance - (1-λ) * max_similarity
    """
    
    selected = []
    remaining = candidates.copy()
    
    while len(selected) < top_k and remaining:
        best_score = -float('inf')
        best_book = None
        
        for book in remaining:
            # 相关性得分
            relevance = book["score"]
            
            # 与已选书籍的最大相似度
            if selected:
                max_similarity = max([
                    calculate_similarity(book, selected_book)
                    for selected_book in selected
                ])
            else:
                max_similarity = 0
            
            # MMR得分
            mmr_score = (lambda_param * relevance - 
                        (1 - lambda_param) * max_similarity)
            
            if mmr_score > best_score:
                best_score = mmr_score
                best_book = book
        
        if best_book:
            selected.append(best_book)
            remaining.remove(best_book)
    
    return selected
```

### 3.4 相似度计算

```python
def calculate_similarity(book1, book2):
    """
    计算两本书的相似度（0-1之间）
    
    维度：
    1. 主题相同：0.4
    2. 难度相同：0.2
    3. 风格相同：0.2
    4. 标签重叠：0.2
    """
    
    similarity = 0.0
    
    # 主题相同
    if book1["topic"] == book2["topic"]:
        similarity += 0.4
    
    # 难度相同
    if book1["difficulty"] == book2["difficulty"]:
        similarity += 0.2
    
    # 风格相同
    if book1["style"] == book2["style"]:
        similarity += 0.2
    
    # 标签重叠（Jaccard相似度）
    tags1 = set(book1["tags"])
    tags2 = set(book2["tags"])
    if tags1 and tags2:
        jaccard = len(tags1 & tags2) / len(tags1 | tags2)
        similarity += 0.2 * jaccard
    
    return similarity
```

## 七、监控指标

### 7.1 四个核心指标

**Pick@1（首屏Top1选择率）**：

```sql
SELECT 
    COUNT(CASE WHEN rank = 1 AND was_clicked = TRUE THEN 1 END) * 100.0 / 
    COUNT(CASE WHEN rank = 1 THEN 1 END) as pick_at_1
FROM recommendations
WHERE created_at > NOW() - INTERVAL '7 days';
```

**解释有用率**：

```sql
SELECT 
    COUNT(CASE WHEN explanation_helpful = TRUE THEN 1 END) * 100.0 / 
    COUNT(*) as explanation_usefulness
FROM recommendation_feedback
WHERE created_at > NOW() - INTERVAL '7 days';
```

**决策时长**：

```sql
SELECT 
    AVG(EXTRACT(EPOCH FROM (rated_at - recommended_at))) as avg_decision_seconds
FROM ratings r
JOIN recommendations rec ON r.book_id = rec.book_id AND r.user_id = rec.user_id
WHERE r.created_at > NOW() - INTERVAL '7 days';
```

**单本均分**：

```sql
SELECT 
    book_id,
    title,
    AVG(rating) as avg_rating,
    COUNT(*) as rating_count
FROM ratings r
JOIN books b ON r.book_id = b.book_id
WHERE r.created_at > NOW() - INTERVAL '30 days'
GROUP BY book_id, title
HAVING COUNT(*) >= 5
ORDER BY avg_rating DESC;
```

# 选书神器候选人试作业文档包 v1.2

## 📋 任务总览（TL;DR）

**目的**：在 3–6 小时内，验证候选人的 **AI 协作开发能力 + 工程落地能力 + 推荐闭环思维**

**核心要求**：

- 完成 `/recommend`、`/feedback`、`/rerank` 三个接口 + 一个"评分页"前端
- 跑通 **首推→逐本打分→再推荐** 的完整闭环
- **≥50% 代码由 AI 辅助生成**，需附详细AI使用记录
- 交付风格：**轻量可运行 > 花哨**，必须可在线访问或本地一键启动

------

## 🎯 1. 场景与目标

### 产品背景

我们做"选书神器"：为用户输出**1分钟精准推荐卡**（30秒说书 + 30秒案例 + 快速自测），并根据逐本评分即时**再推荐**。

### 核心验证点

1. **内容可解释**：每本书给出结构化推荐理由
2. **能学习偏好**：评分后再推荐顺序变化合理
3. **工程可运行**：可部署/可本地一键启动
4. **AI协作能力**：高效使用AI工具完成开发

### 3.2 POST /feedback - 提交评分

**请求体示例：**

```json
{
  "user_id": "user123",
  "ratings": [
    {
      "book_id": "b001",
      "rating": 4,
      "feedback": "很实用，但希望有更多互联网行业案例"
    },
    {
      "book_id": "b002", 
      "rating": 5,
      "feedback": "完美解决了我的跨部门沟通问题"
    }
  ]
}
```

**响应体示例：**

```json
{
  "status": "success",
  "message": "评分已记录，正在更新推荐偏好"
}
```

### 3.3 POST /rerank - 重新推荐

**请求体示例：**

```json
{
  "user_id": "user123",
  "original_request": { /* 原始推荐请求 */ },
  "exclude_rated": true
}
```

**响应体示例：**

```json
{
  "recommendations": [
    {
      "book_id": "b009",
      "title": "刻意练习", 
      "adjustment_reason": "基于你对实用性的偏好(b001评分4分)和跨部门协作的成功经验(b002评分5分)，推荐技能提升类实用书籍"
    }
  ],

}
```

## 🚨 7. 常见问题

### Q1: 推荐算法需要多复杂？

A: 不需要机器学习，基于规则的匹配算法即可。重点是逻辑清晰、可解释。

### Q2: 前端需要多美观？

A: 功能完整 > 视觉效果。简洁可用即可，不需要花哨动画。

### Q3: 如何体现AI协作？

A: 记录详细的提示词、AI生成过程、人工调整。展示AI如何提升开发效率。

### Q4: 时间不够怎么办？

A: 优先完成核心功能（推荐接口+评分页面），其他功能可以简化。

### Q5: 数据可以虚拟吗？

A: 可以，但要符合真实场景。书籍信息要合理，推荐逻辑要说得通。

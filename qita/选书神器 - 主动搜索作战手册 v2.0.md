## 一、核心策略

### 🎯 总体目标

**14天内找到1-2位合适的全栈工程师，完成10天试用并启动MVP开发**

### 📊 漏斗目标

```
主动搜索：200人（14天累计）
    ↓ (回复率40%)
候选人回复：80人
    ↓ (深度沟通率25%)
深度沟通：20人
    ↓ (发送快筛率40%)
发送快筛表单：8人
    ↓ (完成率70%)
完成快筛：5-6人
    ↓ (通过率60%)
电话面试：3-4人
    ↓ (试作业发送率75%)
发送试作业：2-3人
    ↓ (提交率80%)
提交试作业：2人
    ↓ (通过率50%)
进入10天试用：1-2人
```

### 🔑 核心原则

1. **质量 > 数量**：宁可少联系几个，也要确保高匹配度
2. **速度 > 完美**：快速筛选，不要过度纠结
3. **主动 > 被动**：我们选择候选人，而不是等候选人选我们
4. **数据 > 感觉**：每天记录数据，根据转化率优化策略

## 二、理想工程师画像

### 🎯 核心定位："AI协作全栈战斧"

这位工程师不是普通开发者，而是能在14天MVP冲刺中的"战斧"——精准、高效、能直接投入战斗，并且自带AI引擎加持。

### 1️⃣ 基本信息（弹性标准）

#### 经验年限：**弹性设置，不卡死**

**主要目标**：2-5年

- 成本可控（15-20K）
- 能力匹配（独立开发能力）
- 干劲充足（愿意亲手写代码）

**次要目标**：1-2年（潜力股）

- 学习能力强
- 对AI工具熟练
- 薪资期望低（10-15K）

**特殊目标**：8年以上但薪资期望≤15K

- 如唐毅类型（24年经验，10-15K期望）
- 愿意写代码的资深工程师
- 对创业项目感兴趣

**关键判断标准**（优先级从高到低）：

```
1. ✅ 薪资期望（10-20K范围内）
2. ✅ 工作意愿（愿意写代码 > 想做管理）
3. ✅ 地点时间（长沙+随时到岗 > 外地+需考虑）
4. ✅ 状态（离职 > 在职看机会 > 在职不看）
```

#### 地理位置：**优先级设置**

**第一优先**：长沙本地（林科大/岳麓区/高新区附近）

- 立即联系
- 现场协作效率最高

**第二优先**：长株潭都市圈（株洲/湘潭）

- 1小时通勤可接受
- 询问是否愿意每天通勤或短期住长沙

**可接受**：湖南省内（衡阳/常德/岳阳）

- 需确认搬迁意愿
- 或愿意项目期间住长沙

**谨慎考虑**：外省远程

- 需高频同步承诺（每日站会+进度视频）
- 优先已有远程经验的候选人

#### 当前状态：**优先级排序**

```
🥇 离职-随时到岗（最佳）
🥈 离职-考虑机会（次优）
🥉 在职-看机会-1-2周可到岗（可考虑）
⚠️ 在职-暂不看机会（不联系）
❌ 在职-月内不考虑（不联系）
```

#### A类候选人（最优先）：Python + FastAPI + React/Vue + AI

**必须**：

- ✅ Python熟练（项目经验≥1年）
- ✅ FastAPI实战经验（简历明确提到）
- ✅ React/Vue独立开发能力
- ✅ PostgreSQL/MySQL

**加分**：

- 🌟 推荐系统/协同过滤经验
- 🌟 Cursor/Copilot使用经验
- 🌟 Surprise/scikit-learn库使用

**行动**：立即联系（话术版本A）

------

#### B类候选人（次优先）：Python + Django/Flask + 前端 + AI

**必须**：

- ✅ Python熟练
- ✅ Django/Flask项目经验
- ✅ 前端自保能力（React/Vue）
- ✅ 有AI工具使用迹象

**缺失**：

- ⚠️ 无FastAPI经验（需询问学习意愿）

**沟通话术**：

```
"我看你主要用Django/Flask，我们项目是FastAPI，但如果你对异步框架感兴趣，FastAPI学习曲线很平缓，AI协作工具（Cursor）可以大幅降低学习成本。方便聊聊吗？"
```

**行动**：联系并询问学习意愿

------

#### C类候选人（可考虑）：Java/Go + Python副业 + 有AI经验

**必须**：

- ✅ 有Python使用经验（即使是副业项目）
- ✅ 有AI工具/大模型应用经验
- ✅ 全栈能力（前后端都能做）

**缺失**：

- ⚠️ 主技术栈不是Python
- ⚠️ 需确认转型意愿

**沟通话术**：

```
"我看你主要做Java/Go，但也有Python经验，还用过[AI工具/LLM]。我们在做AI推荐系统，技术栈是Python + FastAPI。如果你对Python和AI应用感兴趣，想转型的话，我们提供试作业（500元提交即付），可以先体验一下。"
```

**行动**：试探性沟通（话术版本D）

------

#### D类候选人（谨慎评估）：纯Java/C# + 无Python经验

**除非满足以下所有条件，否则不联系**：

- ✅ 强烈学习意愿（简历中体现正在学Python）
- ✅ AI工具熟练（Cursor/Copilot项目经验）
- ✅ 薪资期望合理（≤15K）
- ✅ 全栈能力（前后端都能做）

**关键判断**：

```
有AI工具经验 > 技术栈
愿意学习新技术 > 当前技术栈
```

### 3️⃣ AI协作能力（核心差异点）

这是识别人才的"照妖镜"。他必须是"AI原生代"开发者。

#### 三级筛选标准

**【简历阶段】寻找信号**

**明确关键词（最佳信号）**：

- Cursor
- GitHub Copilot
- JetBrains AI
- Amazon Q
- Gemini Code Assist
- Claude Code
- AI辅助开发
- Prompt Engineering

**隐性信号（需要判断）**：

- GitHub有AI相关项目
- 技术博客提到AI工具
- 项目描述中有"快速开发"、"效率提升10倍"
- 使用LangChain/Dify/LlamaIndex等AI框架

**加分项**：

- 🌟 个人博客写过AI工具使用心得
- 🌟 GitHub有AI辅助开发的项目
- 🌟 简历明确写"AI优先开发"

**【沟通阶段】直接询问**

```
话术：
"你平时用Cursor、Copilot这类AI编程工具吗？"

→ 如果用：
"能具体说说你在什么场景下用？觉得效果如何？"

→ 如果不用：
"我们团队AI协作优先，用Cursor等工具可以提效10倍，你愿意学习吗？"
```

**【快筛阶段】深度评估**

快筛问卷第1题（8分）：

```
请说出你最熟悉的2个AI编程工具，并分别说明：
1. 你用它们解决过什么具体问题？
2. 使用过程中遇到的"坑"是什么？如何解决的？
3. 它们分别适合什么场景，不适合什么场景？
```

评分标准：

- 7-8分：能说出2+工具 + 具体案例 + 踩坑经验 + 场景判断
- 5-6分：能说出2个工具 + 一些使用经验
- 3-4分：只说出工具名称
- 0-2分：不了解

### 🎯 风险应对手册

#### 风险1：回复率过低（<30%）

**触发条件**：连续3天回复率<30%

**原因分析**：

```
可能原因：
1. 话术不吸引人
2. 联系的候选人不活跃
3. 发送时间不对
4. 候选人收到太多招聘信息（疲劳）
```

**应对措施**：

```
立即行动：
□ 检查话术版本效果（查看工作表3）
□ 切换到效果最好的话术
□ 调整发送时间（试试上午10-11点）
□ 优先联系"在线"状态的候选人

优化方向：
□ 话术更个性化（提到具体项目）
□ 缩短话术长度（控制在150字内）
□ 加强行动号召（直接约时间）
□ 提供项目文档链接（降低沟通成本）
```

------

#### 风险2：高匹配候选人流失

**触发条件**：90%匹配的候选人超过24小时未回复或回复后消失

**原因分析**：

```
可能原因：
1. 回复不够快（被竞争对手抢走）
2. 项目信息不够清晰
3. 没有主动约时间
4. 候选人有其他offer
```

**应对措施**：

```
预防措施：
□ 高匹配候选人优先处理（30分钟内回复）
□ 主动约具体时间（不要说"过几天"）
□ 发送完整项目文档（降低信息不对称）
□ 微信加好友（多一个联系渠道）

补救措施（如果已流失）：
□ 立即电话联系（如果有电话）
□ 发送第三次消息（展示诚意）
□ 提高报价（如果对方有其他offer）
□ 接受：不是每个候选人都能抓住
```

------

#### 风险3：快筛完成率低（<50%）

**触发条件**：发送5份快筛表单，只有2人完成

**原因分析**：

```
可能原因：
1. 问卷太长（5问，10-15分钟）
2. 问题太难
3. 没有激励
4. 候选人太忙
```

**应对措施**：

```
立即优化：
□ 简化问卷（5问 → 3问）
□ 降低难度（减少字数要求）
□ 提供激励（完成送技术资料/小红包）
□ 发送提醒（24小时未完成提醒）

沟通话术：
"[候选人名字]，快筛问卷需要10分钟，完成后我送你一份[价值X元的技术资料/100元红包]，感谢你的时间。"
```

------

#### 风险4：候选人池枯竭

**触发条件**：连续3天每天联系<10人

**原因分析**：

```
可能原因：
1. 长沙IT人才池有限
2. 搜索关键词太精准
3. 筛选条件太严格
```

**应对措施**：

```
扩大范围：
□ 放宽经验要求（1-10年都看）
□ 放宽技术栈（Java+Python副业也可以）
□ 放宽地点（湖南省内都看）
□ 放宽活跃度（一个月内活跃都看）

调整策略：
□ 从策略一（精准）切换到策略二（扩大）
□ 增加策略四（转型候选人）的比重
□ 尝试拉勾网/猎聘网等其他平台
□ 发布JD，让候选人主动来（被动转主动）
```

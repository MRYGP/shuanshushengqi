## 01｜项目速览（TL;DR）

### 核心定位

- **我们做什么**：AI 驱动的职场书籍推荐系统，流程闭环：**画像 → 推荐 → 评分 → 再推荐 → 看板**。
- **核心价值**：每本书不仅被"推荐"，还给出**50/150字的个性化理由**与**Top1 的"30秒案例 + 两步行动"**，可直接落地。
- **用户与模式**：B2B2C（**镜像博主 → 其粉丝**）与 C 端职场人并行验证。

### 当前阶段

- **MVP 冲刺**：**14 天**上线首版，长沙现场优先
- **团队规模**：创始人 + 第一位技术合伙人（你）
- **资金状况**：已完成产品设计和战略规划，预算充足支持6-9个月运营

### 核心指标（口径固定）

- **Pick@1 ≥45%**：首屏Top1被选占比
- **解释有用率 ≥80%**：用户认为推荐理由有用的占比
- **决策时长 ≤60s**：从展示到提交评分的中位时长

### 技术栈

- **后端**：FastAPI / Python + PostgreSQL + Redis
- **前端**：React/Next.js + Tailwind CSS
- **算法**：Surprise（SVD/KNNBaseline）+ **MMR 去相似** + 规则打分
- **监控**：PostHog + Metabase + Sentry

### 长期愿景

把"解释 + 决策"的推荐闭环，迁移为**游戏化的认知训练**：让"经历 → 反馈 → 复盘"成为 AI 时代的默认学习范式。

### 核心算法详解

#### 1. 规则打分

```python
def calculate_match_score(user_profile, book):
    """
    基于用户画像和书籍特征计算匹配分数
    """
    score = 0.0
    
    # 主题匹配 (50%)
    if book.topic in user_profile.topics:
        score += 0.5
    
    # 难度匹配 (30%)
    if book.difficulty == user_profile.difficulty:
        score += 0.3
    elif abs(difficulty_levels.index(book.difficulty) - 
             difficulty_levels.index(user_profile.difficulty)) == 1:
        score += 0.15  # 相邻难度半分
    
    # 风格匹配 (20%)
    if book.style in user_profile.styles:
        score += 0.2
    
    # 时长匹配 (加分项)
    if user_profile.min_minutes <= book.minutes <= user_profile.max_minutes:
        score += 0.1
    
    # 场景匹配 (加分项)
    scenario_overlap = set(book.scenarios) & set(user_profile.scenarios)
    if scenario_overlap:
        score += 0.1 * (len(scenario_overlap) / len(user_profile.scenarios))
    
    return min(score, 1.0)  # 最高1.0
```

#### 2. MMR去相似

```python
def mmr_rerank(candidates, lambda_param=0.7, k=3):
    """
    Maximal Marginal Relevance去相似算法
    
    Args:
        candidates: 候选书籍列表（已按相关性排序）
        lambda_param: 相关性vs多样性权衡参数（0-1，推荐0.7）
        k: 返回数量
    
    Returns:
        去相似后的Top-K书籍
    """
    selected = []
    remaining = candidates.copy()
    
    # 选择第一本（相关性最高）
    selected.append(remaining.pop(0))
    
    while len(selected) < k and remaining:
        mmr_scores = []
        
        for book in remaining:
            # 相关性得分
            relevance = book.match_score
            
            # 与已选书籍的最大相似度
            max_similarity = max(
                calculate_similarity(book, s) for s in selected
            )
            
            # MMR分数
            mmr_score = lambda_param * relevance - (1 - lambda_param) * max_similarity
            mmr_scores.append((book, mmr_score))
        
        # 选择MMR分数最高的
        best_book, _ = max(mmr_scores, key=lambda x: x[1])
        selected.append(best_book)
        remaining.remove(best_book)
    
    return selected

def calculate_similarity(book1, book2):
    """
    计算两本书的相似度（基于多个维度）
    """
    similarity = 0.0
    
    # 主题相似度
    if book1.topic == book2.topic:
        similarity += 0.4
    
    # 作者相似度
    if book1.author == book2.author:
        similarity += 0.2
    
    # 标签重叠度
    tag_overlap = len(set(book1.tags) & set(book2.tags))
    tag_union = len(set(book1.tags) | set(book2.tags))
    if tag_union > 0:
        similarity += 0.3 * (tag_overlap / tag_union)
    
    # 难度相似度
    difficulty_diff = abs(
        difficulty_levels.index(book1.difficulty) - 
        difficulty_levels.index(book2.difficulty)
    )
    similarity += 0.1 * (1 - difficulty_diff / len(difficulty_levels))
    
    return similarity
```

#### 3. 协同过滤（Surprise库）

```python
from surprise import SVD, KNNBaseline, Dataset, Reader
from surprise.model_selection import cross_validate

def train_collaborative_filtering(ratings_data):
    """
    训练协同过滤模型
    """
    # 准备数据
    reader = Reader(rating_scale=(1, 5))
    data = Dataset.load_from_df(
        ratings_data[['user_id', 'book_id', 'rating']], 
        reader
    )
    
    # 训练SVD模型
    algo = SVD(n_factors=50, n_epochs=20, lr_all=0.005, reg_all=0.02)
    cross_validate(algo, data, measures=['RMSE', 'MAE'], cv=5, verbose=True)
    
    # 训练完整数据集
    trainset = data.build_full_trainset()
    algo.fit(trainset)
    
    return algo

def hybrid_recommendation(user_id, rule_based_scores, cf_model, alpha=0.7):
    """
    混合推荐：规则打分 + 协同过滤
    
    Args:
        user_id: 用户ID
        rule_based_scores: 规则打分结果 [(book_id, score), ...]
        cf_model: 协同过滤模型
        alpha: 规则打分权重（推荐0.7，协同过滤0.3）
    
    Returns:
        混合后的推荐列表
    """
    hybrid_scores = []
    
    for book_id, rule_score in rule_based_scores:
        # 获取协同过滤预测
        cf_pred = cf_model.predict(user_id, book_id)
        cf_score = (cf_pred.est - 1) / 4  # 归一化到0-1
        
        # 混合得分
        final_score = alpha * rule_score + (1 - alpha) * cf_score
        hybrid_scores.append((book_id, final_score))
    
    # 按混合得分排序
    hybrid_scores.sort(key=lambda x: x[1], reverse=True)
    
    return hybrid_scores
```

## 06｜看板与指标（固定口径）

### 核心指标定义

#### 1. Pick@1（首选准确率）

```yaml
定义: 首屏Top1被用户选择（点击/评分≥4星）的占比

计算公式:
  Pick@1 = (Top1被选次数) / (总推荐次数)

目标值: ≥45%

分层统计:
  - 按用户画像（岗位/年限）
  - 按主题分类
  - 按推荐算法
  - 按时段（工作日/周末）

AB测试场景:
  - 3本 vs 5本推荐
  - 50字 vs 150字理由
  - 不同探索比（10%/20%/30%）
```

#### 2. 解释有用率

```yaml
定义: 用户认为推荐理由有用的占比

计算方式:
  方式1: 用户主动标记"理由有用"
  方式2: 评分≥4星时视为理由有效

计算公式:
  解释有用率 = (理由有用数) / (总推荐数)

目标值: ≥80%

优化方向:
  - 理由个性化程度
  - 案例相关性
  - 行动步骤可操作性
  - 语言表达清晰度
```

#### 3. 决策时长

```yaml
定义: 从展示推荐结果到提交评分的中位时长

计算公式:
  决策时长 = median(提交时间 - 展示时间)

目标值: ≤60秒

监控维度:
  - P50（中位数）
  - P90（90分位）
  - P99（99分位）

影响因素:
  - 推荐数量（3本 vs 5本）
  - 理由长度（50字 vs 150字）
  - UI交互设计
  - 用户熟悉度
```

#### 4. 单本均分

```yaml
定义: 被选择书籍的平均星级

计算公式:
  单本均分 = sum(所有评分) / count(评分次数)

目标值: ≥4.0

细分统计:
  - 按书籍分类
  - 按难度级别
  - 按用户群体
  - 按推荐算法
```

## 08｜你将面对的真实挑战

### 技术挑战

#### 1. 冷启动问题

```yaml
场景: 新用户没有历史数据，如何精准推荐？

解决方案:
- 方案1: 精心设计的画像问卷（3分钟内完成）
- 方案2: 基于岗位和年限的模板推荐
- 方案3: 探索性推荐（10-30%）
- 方案4: 快速学习机制（3次交互后调整）

技术难点:
- 问卷设计的平衡（信息量 vs 用户体验）
- 模板推荐的准确性
- 探索比的动态调整
- 快速学习算法

你需要做的:
- 实现多种冷启动策略
- AB测试不同方案效果
- 监控冷启动用户的转化率
- 持续优化问卷和算法
```

#### 2. 数据稀疏性

```yaml
场景: 初期评分数据少，协同过滤效果差

解决方案:
- 方案1: 规则打分为主（70%）+ 协同过滤为辅（30%）
- 方案2: 利用隐式反馈（点击/浏览时长）
- 方案3: 迁移学习（利用其他推荐系统的数据）
- 方案4: 渐进式权重调整

技术难点:
- 混合推荐的权重动态调整
- 隐式反馈的噪声处理
- 数据不足时的效果评估
- 过拟合风险控制

你需要做的:
- 实现混合推荐引擎
- 设计数据积累策略
- 监控数据量与效果的关系
- 建立自动化权重调整机制
```

#### 3. 解释质量

```yaml
场景: 生成的推荐理由流于表面，无法打动用户

核心要求:
- 不是复述书籍简介
- 而是说明"为什么适合你"
- 结合用户画像和当前挑战
- 提供具体的应用场景

技术实现:
- 方式1: 模板 + 规则生成
- 方式2: Claude API生成 + 人工审核
- 方式3: 预训练模型微调

质量标准:
- 个性化程度（提到用户具体场景）
- 可操作性（有明确的行动建议）
- 说服力（有案例或数据支撑）
- 简洁度（50/150字限制）

你需要做的:
- 实现理由生成引擎
- 建立质量评估体系
- AB测试不同生成方式
- 持续优化模板和提示词
```

#### 4. 多样性与准确性

```yaml
场景: 推荐太相似缺乏新意，太多样化又不够精准

平衡策略:
- MMR算法：λ参数调优（0.6-0.8）
- 分层推荐：Top1保证准确，Top2-3增加多样性
- 主题覆盖：确保不同主题都有覆盖
- 难度梯度：入门 → 进阶 → 专业合理分布

监控指标:
- Top3重复率（<33%为优）
- 主题分布均衡度
- 难度分布合理性
- 用户满意度

你需要做的:
- 实现MMR算法
- 可视化多样性指标
- 提供λ参数调优工具
- AB测试最优参数组合
```

## 15｜FAQ（完整版）

### 产品相关

**Q1: 这不就是"做书单"的工具吗？**

A: 不是。我们强调**可解释推荐**和**再推荐学习**闭环。区别在于：

- 书单工具：给你一个列表，不解释为什么
- 我们：每本书都有50/150字的个性化理由，说明"为什么适合你"
- 书单工具：推荐后就结束了
- 我们：基于你的反馈（星级+标签）智能调整再推荐
- 书单工具：静态的、一次性的
- 我们：动态的、持续学习的推荐系统

**Q2: 为什么先做书，后做学习？**

A: 书籍是最佳的冷启动选择：

1. **结构化**：知识密度高，易于标准化
2. **可验证**：用户反馈明确（读了/没读，有用/没用）
3. **低风险**：推荐错了成本低（读另一本即可）
4. **高频次**：职场人每月读1-3本书，迭代快
5. **成熟生态**：已有豆瓣/微信读书等平台，用户有认知

闭环跑通后，再升级为"经历驱动"的学习关卡：

- 方法论 → 情境对话
- 阅读 → 决策体验
- 知识 → 能力培养

**Q3: 和ChatGPT推荐书有什么区别？**

A:

- ChatGPT：通用对话，无专业数据，碎片化，无系统

### 技术相关

**Q5: 为什么不用深度学习做推荐？**

A: MVP阶段选择"规则+协同过滤"的原因：

1. **数据量不足**：深度学习需要大量数据，我们初期没有
2. **可解释性**：规则打分逻辑清晰，易于调优和解释
3. **冷启动**：规则可以很好处理新用户
4. **开发效率**：Surprise库成熟稳定，3周可上线
5. **成本考虑**：规则+协同过滤的计算成本低

后续会：

- 数据积累后逐步引入深度学习
- 使用LightFM/DeepFM等混合模型
- 但依然保持规则兜底机制

**Q6: MMR算法为什么重要？**

A: MMR（Maximal Marginal Relevance）解决多样性问题：

问题：纯相关性排序可能推荐3本非常相似的书 例如：都是沟通类的书，主题、难度、风格都类似

MMR作用：

- 平衡相关性和多样性
- Top1保证最相关
- Top2-3增加多样性
- 用户获得更丰富的选择

效果：

- 降低Top3重复率（<33%）
- 提升用户满意度
- 增加探索性

**Q7: 如果协同过滤效果不好怎么办？**

A: 我们有多层兜底机制：

1. **混合推荐**：规则70% + 协同过滤30%
2. **降级策略**：协同过滤失败自动回退到规则
3. **AB测试**：持续验证不同权重组合
4. **监控告警**：实时监控推荐效果
5. **数据积累**：随着数据增加逐步提升权重

保证：无论协同过滤效果如何，都能提供基础推荐

### 核心技术栈选择（及理由）

**前端：React + TypeScript**

- 为什么：组件化开发，后续扩展容易
- 学习成本：你应该熟悉，如果不熟悉，1周可以上手
- 风险：低，生态成熟

**后端：FastAPI + Python**

- 为什么：
    - Python适合快速开发和AI集成
    - FastAPI性能好，开发效率高
    - 异步支持，适合大量API调用
- 学习成本：如果你熟悉Django/Flask，几乎0成本
- 风险：低，文档齐全

**数据库：PostgreSQL**

- 为什么：
    - 开源、稳定、功能强大
    - 支持JSON字段（灵活存储用户画像）
    - 可以后续扩展到图数据库
- 学习成本：如果你熟悉MySQL，区别不大
- 风险：低

**AI服务：Claude API / GPT-4**

- 为什么：
    - 不需要自己训练模型
    - API调用简单
    - 成本可控（0.3-1元/次对话）
- 学习成本：几小时就能掌握API调用
- 风险：中（依赖第三方服务，但有备选方案）

**缓存：Redis**

- 为什么：
    - 减少重复AI调用
    - 提升响应速度
- 学习成本：如果你用过，0成本；如果没用过，1天上手
- 风险：低

#### 模块2：推荐引擎

**功能：**

- 基于用户画像推荐书籍
- 递进式路径规划
- 加入10-30%随机性（探索推荐）

**技术实现：**

```python
# 推荐算法流程
def recommend_books(user_profile, num_books=3):
    # Step 1: 确定用户当前能力水平
    current_level = calculate_level(user_profile)
    
    # Step 2: 筛选适合的书籍
    candidate_books = filter_books(
        min_level=current_level - 1,
        max_level=current_level + 1,
        interests=user_profile['interests']
    )
    
    # Step 3: 计算匹配度
    for book in candidate_books:
        book.score = calculate_match(user_profile, book)
    
    # Step 4: 加入随机性
    # 70%高匹配度书籍 + 20%跨界书籍 + 10%意外发现
    recommendations = []
    recommendations.extend(
        sorted(candidate_books, key=lambda x: x.score)[:2]  # 70%
    )
    recommendations.append(
        random.choice(cross_domain_books)  # 20%
    )
    recommendations.append(
        random.choice(all_books)  # 10%
    )
    
    return recommendations
```

**技术挑战：**

- 如何定义"递进式"？
    - 方案：建立书籍依赖图（A书是B书的前置）
- 随机性如何控制？
    - 方案：用户反馈训练，动态调整比例
- 如何避免推荐重复？
    - 方案：记录推荐历史，去重

**工作量估算：3周**

- Week 1: 基础推荐算法

**Q3: 期权怎么兑现？** A: 4年行权期，1年cliff。公司被收购或上市时可以兑现。

**Q4: 工作强度会很大吗？** A: 创业初期确实辛苦，但我们不鼓励无效加班。关键是效率，不是时长。

**Q5: 如果我们合作不愉快怎么办？** A: 前3个月是磨合期，任何一方都可以无条件退出。但我相信我们能合作好。

## 第一部分：项目背景与愿景

### 我们要解决什么问题？

**问题场景**： 一个工作3年的产品经理想提升沟通能力：

- 打开豆瓣：10万条书评，不知道哪本适合自己
- 问ChatGPT：给一个通用书单，没有个性化理由
- 看小红书：博主推荐很主观，不知道是否适合自己的职业阶段

**核心痛点**：

1. ❌ 推荐不精准（不考虑职业、经验、学习风格）
2. ❌ 理由不清晰（不知道"为什么"推荐这本书）
3. ❌ 无法学习偏好（推荐后没有反馈闭环）

### 我们的解决方案

**双维度推荐引擎**：

```
维度1：需求匹配（你的职业困境 × 书籍能解决的问题）
维度2：偏好匹配（你的学习风格 × 书籍表达风格）
```

**可解释性设计**： 每本书的推荐卡包含：

- 50字快速了解：书的核心观点
- 50字为什么适合你：结合你的画像说明
- 50字如何应用：两步行动建议
- 预估阅读时长：15-600分钟分级

**学习闭环**：

```
首推（3-5本） 
    ↓ 
用户打分+标签（"太理论"/"很实操"/"故事强"）
    ↓ 
权重调整+再推荐（Top3必有变化）
    ↓ 
数据看板分析（哪些书被选择？为什么？）
    ↓ 
算法优化（持续提升准确率）
```

### 核心技术挑战与方案

#### 挑战1：冷启动问题

**问题**：新用户没有历史数据，协同过滤无法工作

**解决方案**：

```python
def get_recommendations(user_profile, user_id=None):
    # 策略1：规则打分（基于画像）- 权重60%
    rule_scores = rule_based_scorer(user_profile)
    
    # 策略2：协同过滤（如果有历史） - 权重40%
    if user_id and has_history(user_id):
        cf_scores = collaborative_filter(user_id)
    else:
        # 冷启动兜底：流行度 + 探索
        cf_scores = popularity_based(
            explore_ratio=0.2  # 20%探索新书
        )
    
    # 混合推荐
    final_scores = 0.6 * rule_scores + 0.4 * cf_scores
    
    # MMR去相似
    diverse_recs = mmr_rerank(final_scores, lambda=0.7)
    
    return diverse_recs
```

#### 挑战3：MMR多样性平衡

**问题**：如何在"准确"和"多样"之间找平衡？

**算法核心**：

```python
def mmr_rerank(candidates, lambda_param=0.7, top_k=3):
    """
    MMR算法：Maximal Marginal Relevance
    
    Score(d) = λ * Relevance(d) - (1-λ) * max[Similarity(d, selected)]
    
    λ = 1.0 : 完全看相关性（可能推荐3本同类书）
    λ = 0.0 : 完全看多样性（可能推荐不相关的书）
    λ = 0.7 : 平衡点（推荐准确且有一定多样性）
    """
    selected = []
    remaining = candidates.copy()
    
    while len(selected) < top_k:
        best_score = -float('inf')
        best_item = None
        
        for item in remaining:
            # 相关性得分
            relevance = item['match_score']
            
            # 与已选书籍的最大相似度
            if selected:
                similarity = max([
                    get_similarity(item, s) for s in selected
                ])
            else:
                similarity = 0
            
            # MMR得分
            mmr_score = lambda_param * relevance - (1 - lambda_param) * similarity
            
            if mmr_score > best_score:
                best_score = mmr_score
                best_item = item
        
        selected.append(best_item)
        remaining.remove(best_item)
    
    return selected
```

**参数调优策略**：

```
测试组1：λ=0.9（偏准确）→ Top3重复率？用户满意度？
测试组2：λ=0.7（平衡）  → 基准组
测试组3：λ=0.5（偏多样）→ 探索效果？意外惊喜率？
```

#### 挑战4：评分后的再推荐

**问题**：如何把用户的标签反馈（"太理论""很实操"）转化为权重调整？

**映射逻辑**：

```python
# 标签→书籍特征映射表
TAG_MAPPING = {
    "太理论": {"theory_depth": -0.3, "tool_density": +0.2},
    "很实操": {"tool_density": +0.3, "story_ratio": +0.1},
    "故事强": {"story_ratio": +0.3, "theory_depth": -0.1},
    "案例丰富": {"case_count": +0.3},
}

def rerank_with_feedback(user_ratings):
    """
    基于评分和标签调整推荐权重
    """
    weight_adjustments = defaultdict(float)
    
    for rating in user_ratings:
        book = rating['book']
        score = rating['score']  # 1-5星
        tags = rating['tags']    # ["太理论", "很实操"]
        
        # 正向强化（高分书的特征）
        if score >= 4:
            for feature, value in book['features'].items():
                weight_adjustments[feature] += 0.1 * (score - 3)
        
        # 标签修正（无论评分高低）
        for tag in tags:
            for feature, adjustment in TAG_MAPPING[tag].items():
                weight_adjustments[feature] += adjustment
    
    # 使用调整后的权重重新推荐
    return recommend_with_weights(weight_adjustments)
```

### 数据看板设计

#### 四个核心指标

**1. Pick@1（首屏Top1选择率）≥ 45%**

```
定义：用户最终选择的书，在首推Top3中排第1的比例
计算：(Top1被选次数 / 总选择次数) × 100%

为什么重要：
- 衡量推荐精准度的最直接指标
- Pick@1高 → 我们的Top1推荐很准
- Pick@1低 → 排序有问题，需要调整权重
```

**2. 解释有用率 ≥ 80%**

```
定义：用户认为"推荐理由有用"的比例
收集：评分页有"这个推荐理由有用吗？"👍/👎

为什么重要：
- 衡量可解释性的关键指标
- 高有用率 → 用户理解为什么推荐
- 低有用率 → 理由模板需要优化
```

**3. 决策时长 ≤ 60s**

```
定义：从看到推荐到完成评分的平均时长
收集：PostHog埋点 (view_recommendation → submit_rating)

为什么重要：
- 决策时长短 → 推荐清晰易懂
- 决策时长长 → 用户犹豫，信息过载
```

**4. 单本均分（辅助指标）**

```
定义：每本书的平均评分
收集：用户1-5星评分

用途：
- 识别"人人都喜欢"的书（基准书）
- 识别"评价两极"的书（需要精准匹配）
- 调整流行度权重
```

```python
def rule_based_scorer(user_profile, book):
    """
    基于用户画像和书籍特征打分
    """
    score = 0.0
    
    # 主题匹配（权重0.5）
    if book['topic'] in user_profile['topics']:
        score += 0.5
    
    # 难度匹配（权重0.3）
    if book['difficulty'] == user_profile['difficulty']:
        score += 0.3
    elif abs(DIFFICULTY_MAP[book['difficulty']] - 
             DIFFICULTY_MAP[user_profile['difficulty']]) == 1:
        score += 0.15  # 相邻难度打半分
    
    # 风格匹配（权重0.2）
    if book['style'] in user_profile['styles']:
        score += 0.2
    
    # Bonus：场景匹配
    if any(s in book['scenarios'] 
           for s in user_profile['scenarios']):
        score += 0.1
    
    return min(score, 1.0)  # 最高1.0分
```
